# Roshambo 游戏需求文档 (V2.1)

## 1. 项目概述
《Roshambo》是一款基于 HTML5 + CSS3 + TypeScript 开发的 Roguelike 卡牌游戏。玩家通过向 3x3 矩阵插入卡牌，通过石头剪刀布的克制关系进行博弈并冲击高分。

## 2. 核心数据类型
### 2.1 基础元素 (RPS)
- **ROCK (石头)**, **SCISSORS (剪刀)**, **PAPER (布)**, **BLANK (空白)**
- 克制规则：石头 > 剪刀，剪刀 > 布，布 > 石头。

### 2.2 卡牌 (Card)
- 包含 3 个 RPS 元素的数组（1x3 图案）。
- 支持 180 度翻转。
- **资产池限制**：为了确保视觉与底层数据一致，随机生成的卡牌必须且只能从已有的 `KNOWN_ASSETS` 中抽取。游戏理论上有 40 种独特的卡牌组合（排除颠倒重复项），当前已实现 26 种，仍有 14 种缺失。

### 2.3 矩阵 (Matrix)
- 3x3 网格。
- **主题判定**：数量最多的元素为主题。平局优先级：石头 > 剪刀 > 布。

## 3. 核心玩法
### 3.1 记分规则 (Scoring)
- **胜利判定**：新主题克制旧主题，或主题相同时新矩阵中主题元素数量（Power）更高。
- **基础得分**：胜利默认获得 5 分。
- **额外得分**：在基础分之上，旧矩阵中主题元素每多一个，增加 X 分：
  - 石头 (ROCK)：+4 分/个
  - 剪刀 (SCISSORS)：+3 分/个
  - 布 (PAPER)：+1 分/个
- **失败惩罚**：玩家打出的卡牌永久移除，不获得分数。
- **平局奖励**：获得打出的卡牌。

### 3.2 游戏流程
- 移除 Blind 和目标分数设定。
- 玩家将手牌全部打完，目标是获得尽可能高的分数。
- 游戏结束时展示单独的“结束”弹窗。

## 4. 交互与 UI (UI/UX)
### 4.1 整体布局 (Layout)
- **全屏暗黑模式** (`--bg-dark: #000000`)。
- **左侧边栏 (Sidebar)**：展示 Score、VS 战斗面板（当前主题力量 vs 新主题力量）、Shuffle 和 Deal 操作区，以及底部的 End / Rotate 按钮。
- **右侧游玩区 (Play Area)**：包含顶部 Relics 槽位、居中的 3x3 矩阵（含 Drop Zone 与 Preview Box），以及底部呈扇形展开的手牌区。

### 4.2 手牌展示
- 手牌以“小丑牌 (Balatro)”风格在屏幕底部呈扇形展开。
- 选中的手牌会被抬高并高亮显示（`drop-shadow(0 0 15px rgba(255, 152, 0, 0.8))`）。

### 4.3 预览系统 (Preview)
- **抓取逻辑**：点击手牌将其“抓起”。鼠标跟随移动卡牌。
- **吸附预览**：
  - 抓起的牌 hover 在矩阵四边时，自动吸附并预览推演后的新矩阵（主题、数量、获胜状态、得分预测）。
  - **吸附渲染方案**：为了保证与矩阵动态缩放的比例 (`--cell-size` 和 `--cell-gap`) 完美对齐，Drop Zone 的卡牌预览不使用单张大图，而是动态生成一个对应的 1x3 CSS Grid 容器，将内部的三个 `BlockType` 独立渲染。
- **操作确认**：在四边或者 Preview Box 区域点击鼠标以确认放置。
- **翻转同步**：点击 Rotate 按钮，手牌和预览中的卡牌同步翻转 180 度。
- **取消抓取**：点击空白区域放回卡牌。

## 5. 资源规范
- `Sketch/BlockType={xxx}.png`：矩阵方块。
- `Sketch/CardType={xxx}.png`：手牌长条。
  - 编码规则：0=Blank, 1=Paper, 3=Scissors, 4=Rock。
- `Sketch/IconType={xxx}.png`：UI 图标（用于 VS 面板展示当前主题）。

## 6. 问题追溯与参考图 (Reference)
详见 `reference/` 目录。我们修复了以下历史核心 Bug：
- **“披着空牌外衣的剪刀牌”Bug**：
  - **现象**：[bug-000-scissors.png](./reference/bug-000-scissors.png)（明明视觉是 000，却算出了 4 把剪刀）。
  - **根源**：底层生成了带剪刀的卡牌，但 UI 渲染时因找不到对应资产，触发了 Fallback 回退到 `000.png`。
  - **修复**：修改随机逻辑，强制从 `KNOWN_ASSETS` 中抽取。
- **卡牌吸附尺寸不匹配 Bug**：
  - **现象**：[ui-card-size-mismatch.png](./reference/ui-card-size-mismatch.png)（Attach 时的卡牌比矩阵里的小一圈）。
  - **根源**：整张卡牌原图的内部间隙是写死的像素，无法与浏览器流式缩放的 `var(--cell-gap)` 对齐。
  - **修复**：在预览状态下使用 CSS Grid 分拆渲染独立方块。
- **Vite 黑屏崩溃 Bug**：
  - **现象**：[ui-black-screen.png](./reference/ui-black-screen.png)（整个网页黑屏）。
  - **根源**：一方面是由于意外生成的编译缓存（`.js` 文件）污染了 Vite 服务器；另一方面是 UI 挂载逻辑中意外写了 `this.root.appendChild(app)` 导致自我循环嵌套 (`HierarchyRequestError`)。
  - **修复**：清空 `node_modules/.vite` 缓存，移除引发嵌套的 DOM 操作。
