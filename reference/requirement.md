# Roshambo 游戏需求文档 (V3.0 实际实装版)

> **修订说明**：此文档已根据代码实际实装情况进行了同步更新，修正了旧版文档中关于得分机制、关卡流程以及局外成长部分的缺失。

## 1. 项目概述
《Roshambo》是一款基于 HTML5 + CSS3 + TypeScript 开发的 Roguelike 卡牌游戏。玩家通过向动态矩阵的边缘推入卡牌，利用“石头剪刀布”的克制关系触发“轨道对冲 (Lane Clash)”机制进行博弈，并辅以局间的商店经济系统冲击高分。

## 2. 核心数据类型
### 2.1 基础元素 (RPS)
- **ROCK (石头)**, **SCISSORS (剪刀)**, **PAPER (布)**, **BLANK (空白)**
- 基础克制规则：石头 > 剪刀，剪刀 > 布，布 > 石头。
- **元素价值权重 (Score Weights)**：石头=4，剪刀=3，布=1，空白=0。

### 2.2 卡牌与牌库 (Card & Deck)
- **卡牌**：包含 3 个 RPS 元素的数组（1x3 图案），支持 180 度翻转。
- **初始牌组 (Deck Selection)**：游戏开始时（`CHOOSE_DECK` 状态），玩家可以选择 3 种不同风格的牌组（Balanced, Multi-hit, Hardcore），影响牌库中元素的分布。
- **资产池限制**：为了确保视觉与底层数据一致，生成卡牌必须从预定义的 `KNOWN_ASSETS` 中抽取。

### 2.3 动态矩阵 (Matrix)
- 矩阵大小不再固定为 3x3，而是根据当前关卡的**主题 (Theme)** 动态变化：
  - **Pocket**: 2x2 网格
  - **Rubik**: 3x3 网格
  - **Master**: 4x4 网格

## 3. 核心玩法：轨道对冲 (Lane Clash)
> **【重要更新】**：废弃了原有的“计算全局主题数量”的记分方式，改为更具深度的**轨道对冲穿透机制**。

### 3.1 结算逻辑 (Clash Logic)
玩家从矩阵的四个边缘（TOP, BOTTOM, LEFT, RIGHT）选定一个偏移量推入 1x3 的卡牌。卡牌上的 3 个元素分别作为“攻击方”，沿着对应的轨道（行或列）向矩阵内的“防守方”逐格发起冲击：
1. **攻击方胜利**（克制防守方，或防守方为空白）：
   - 攻击方占据该格位置（替换原有元素）。
   - 玩家**获得得分**，增加分数为**防守方元素的价值权重**（如击碎石头得 4 分）。
   - 攻击方继续向轨道的下一格发起冲击，直到平局、失败或穿透整个矩阵。
2. **平局 (Tie)**（双方元素相同）：
   - 攻击方停止前进，矩阵该格保持不变，不产生分数增减。
3. **攻击方失败**（被防守方克制）：
   - 攻击方停止前进并被摧毁。
   - 玩家受到**惩罚倒扣分**，扣除分数为**攻击方自身的价值权重**。
   - **轨道平移 (Lane Shift)**：触发惩罚的同时，该条轨道上的所有元素会向**与推入方向相反的方向**平移一格。

### 3.2 关卡与游戏流程 (Game Flow)
- **关卡结构**：游戏共有 9 个大关 (Stage)，每个大关 3 个小关 (Tier)，总计 27 关。
- **通关条件**：每关有独立的目标分数 (`goal`) 和基础奖励 (`reward`)。玩家需要在手牌打空前，达到目标分数。
- **失败条件**：手牌耗尽且分数未达标，则游戏结束 (`GAME_OVER`)。

## 4. 局外成长与经济系统 (Economy & Relics)
> **【代码已实装，补充说明】**

### 4.1 资源管理
- **筹码 (Chips)**：游戏基础货币。
- **利息 (Interest)**：当前筹码乘以利息率 (`interestRate`)，在每回合结算时发放。
- **操作资源**：
  - `dealsLeft`：替换手牌的次数，允许玩家丢弃选中的手牌并从牌库抽牌。
  - `shufflesLeft`：矩阵洗牌次数，重新随机生成当前矩阵。

### 4.2 商店与特殊牌 (Shop & Special Cards)
- 关卡胜利后会进入结算界面 (`ROUND_REWARD`)，随后进入商店 (`SHOP`)。
- 商店会随机刷出 3 个**特殊牌 (Relics)**，玩家可消耗筹码购买，获取全局被动增益。
- **已实装特殊牌库**：
  - **Chip Cache (Cache)**：普通稀有度，购买立即获得 8 筹码。
  - **Interest Engine (Engine)**：非凡稀有度，被动效果：每个关卡清理后提供额外的 3 点利息预测加成。
  - **Supply Drop (Supply)**：普通稀有度，被动效果：每个关卡开始时额外提供 +1 替换手牌次数和 +1 洗牌次数。

## 5. 交互与 UI (UI/UX)
- 整体采用全屏暗黑模式 (`--bg-dark: #000000`)。
- **吸附预览 (Preview Box)**：支持在矩阵边缘悬停实时推演并预览对冲结果（得分增减、轨道平移效果），使用动态 CSS Grid 生成分离的预览方块，完美对齐网格间隙。
- **手牌扇形展开**：底部手牌区支持动态计算卡牌数量，进行等比例扇形展开（基于 GSAP 动画）。支持鼠标滚轮或 Rotate 按钮翻转手牌。
